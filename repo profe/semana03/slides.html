What is NaN?

NaN stands for "Not a Number". It is how pandas marks a missing value: a cell where no data exists.

Think of NaN like an empty seat in a classroom. The seat exists, but nobody is sitting there. You need to decide: leave it empty, or assign someone to it?

Detecting Missing Values

The Code

# How many missing per column? df.isnull().sum() # Percentage missing per column (df.isnull().sum() / len(df)) * 100 # Total missing in entire dataset df.isnull().sum().sum()

What Each Part Does

isnull()

Returns True/False for every cell. True = missing.

.sum()

Counts the True values (because True = 1).

.sum().sum()

First sum counts per column, second sum totals everything.

Pro tip: Always check missing values FIRST before any analysis. It's the #1 step in every cleaning workflow.

When to Drop vs. When to Fill?

This is the most important decision in data cleaning. There is no single right answer. It depends on how much is missing and what the data means.

How much of the column is missing?
↓
> 50% missing
Drop the column
Too little data to be useful
< 5% missing
Drop the rows
Safe: tiny data loss
5-50% missing
Fill with a value
Use 0, mean, or median
Key question: Is this column essential for your analysis? If yes, fill it. If not, consider dropping it.

Removing Missing Values: dropna()

dropna() removes rows (or columns) that contain missing values. Use it when the missing data is rare or the row is useless without the value.

# Drop rows with ANY missing value (aggressive!) df_clean = df.dropna() # Drop rows missing a SPECIFIC column (safer) df_clean = df.dropna(subset=['department']) # Drop rows only if ALL values are missing df_clean = df.dropna(how='all') # Drop COLUMNS with missing (axis=1) df_clean = df.dropna(axis=1)

When to use dropna()

< 5% of rows are affected

The missing column is critical (like a name or ID)

You have plenty of data to spare

When NOT to use dropna()

Many rows have missing values

You'd lose too much data

The missing pattern is not random

Filling Missing Values: fillna()

fillna() replaces NaN with a value you choose. The right replacement depends on what the column represents.

# Fill with a constant value df['students'] = df['students'].fillna(0) # Fill with the column average df['dropout_rate'] = df['dropout_rate'].fillna(df['dropout_rate'].mean()) # Fill with the column median (robust to outliers) df['score'] = df['score'].fillna(df['score'].median()) # Fill text columns with a label df['region'] = df['region'].fillna('Unknown')

Fill with 0

Counts, totals

"Not reported = zero"

Fill with mean

Symmetric data

"Use the average"

Fill with median

Skewed data

"Use the middle"

Fill with label

Text columns

"Unknown"

Common mistake: df['col'].fillna(0) does NOT change the original DataFrame. You must assign the result: df['col'] = df['col'].fillna(0)

What Are Data Types?

Every column in a DataFrame has a data type that tells pandas how to store and process the values.

Think of it like putting things in the right boxes. If you put a book in the "food" box, the kitchen staff won't know what to do with it. Similarly, if a number is stored as text, pandas can't do math with it.
pandas Type What It Stores Example Can Do Math?
int64 Whole numbers 42, 2020, 1500 Yes
float64 Decimal numbers 3.14, 99.5, 0.0 Yes
object Text (strings) "Bogota", "2020" No
datetime64 Dates and times 2020-01-15 Yes (differences)
bool True / False True, False Yes (True=1)
The #1 type problem: A column looks like numbers ("2020", "1500") but pandas loaded it as object (text). You can't calculate the mean of text!

Converting Data Types

Check current types

# See all column types df.dtypes # Example output: # name object (text) # year float64 (decimal?) # students object (text!) # active object (text!)
Tip: Run df.dtypes right after loading. If a column you expect to be numeric says object, it needs conversion.

Fix them

# Convert text to integer df['year'] = df['year'].astype(int) # Convert text to decimal df['price'] = df['price'].astype(float) # Convert to date df['date'] = pd.to_datetime(df['date']) # Convert to category (saves memory) df['region'] = df['region'].astype('category')
Trap: astype(int) crashes if the column has NaN values. Always fill or drop NaN before converting to int.

Safe pattern: df['year'] = df['year'].fillna(0).astype(int)

What Are Duplicates?

A duplicate is a row that appears more than once with identical values.

Imagine counting students in a classroom. If you accidentally count Maria twice, your total is wrong. Duplicates in data do the same thing: they inflate your counts and distort your averages.

Removing Duplicates

# How many duplicates exist? df.duplicated().sum() # See the duplicate rows df[df.duplicated(keep=False)] # Remove exact duplicates (keep first occurrence) df = df.drop_duplicates() # Remove duplicates based on specific columns df = df.drop_duplicates(subset=['school', 'year'])

duplicated()

Returns True/False for each row. True means "this row appeared before."

keep=False marks ALL copies as True (not just the second one).

drop_duplicates()

Removes the duplicate rows, keeping the first occurrence by default.

subset= lets you check only specific columns for duplication.

Be careful: Not all duplicates are errors. A school reporting the same data for two different grades may look duplicated but is valid. Use subset= to check only the columns that should be unique.
