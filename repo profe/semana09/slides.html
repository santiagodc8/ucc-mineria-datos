Static vs Interactive: The Evolution

Static Charts (Matplotlib)

üì∏
  • One fixed view of data
  • Can't zoom or pan
  • No hover information
  • New question = new chart
  • Good for: Reports, publications

Interactive Charts (Plotly)

üé¨
  • User explores the data
  • Zoom, pan, select regions
  • Hover shows exact values
  • Filter without new code
  • Good for: Dashboards, exploration

Key Insight: Static charts TELL, interactive charts INVITE exploration

Static chart = Museum painting: "Look, but don't touch"
Interactive chart = Video game: "Here are the controls. Go explore!"

Plotly: Your Interactive Chart Library

What is Plotly?

A Python library that creates interactive charts for the web.

If Matplotlib is a camera (takes static photos),
Plotly is a video camera (captures dynamic content)
Key Advantage: Plotly charts work in Jupyter notebooks, web apps, and dashboards with ZERO extra configuration.

Built-in Interactivity

  • Hover: See exact values on mouse-over
  • Zoom: Click and drag to zoom in
  • Pan: Move around the zoomed view
  • Select: Box or lasso select points
  • Download: Save as PNG with one click
  • Compare: Toggle traces on/off
# Installation (run once in terminal) pip install plotly # Import in your code import plotly.express as px

Plotly: Interactive Scatter Plot

import plotly.express as px import pandas as pd # Load budget data df = pd.read_csv('budget_execution.csv') # Create interactive scatter plot fig = px.scatter( df, x='budget_approved', y='budget_executed', color='department', size='execution_rate', hover_name='project_name', hover_data=['year', 'category'], title='Budget: Approved vs Executed' ) fig.show()

Key Parameters

ParameterWhat it does
x, yPosition on axes
colorColor by category
sizePoint size by value
hover_nameMain hover label
hover_dataExtra hover info
Try it: Hover over points to see project details. Click legend items to show/hide departments.

One line of code = interactive chart with zoom, hover, and filtering

Plotly: Interactive Bar Chart

# Basic bar chart fig = px.bar( df, x='department', y='budget_executed', color='category', title='Budget Executed by Department' ) fig.show() # Grouped bar chart fig = px.bar( df, x='department', y='budget_executed', color='year', barmode='group', # Side by side title='Budget by Year' ) fig.show() # Horizontal bar chart fig = px.bar( df, x='budget_executed', y='department', orientation='h', title='Horizontal View' ) fig.show()

Bar Mode Options

barmode='stack'

Bars stacked on top of each other. Good for: total composition

barmode='group'

Bars side by side. Good for: comparing categories

orientation='h'

Horizontal bars. Good for: long category names

Customizing Plotly Charts

# Create figure fig = px.scatter( df, x='budget_approved', y='budget_executed', color='department' ) # Customize layout fig.update_layout( title={ 'text': 'Budget Analysis', 'font_size': 24, 'x': 0.5 # Center title }, xaxis_title='Approved (COP)', yaxis_title='Executed (COP)', legend_title='Department', template='plotly_white' ) # Add reference line fig.add_shape( type='line', x0=0, y0=0, x1=df['budget_approved'].max(), y1=df['budget_approved'].max(), line=dict(dash='dash', color='gray') ) fig.show()

Available Templates

  • plotly - Default blue background
  • plotly_white - Clean white
  • plotly_dark - Dark mode
  • ggplot2 - R-style
  • seaborn - Seaborn-style
  • simple_white - Minimal
Pro Tip: Add a diagonal reference line (y=x) to show where executed equals approved budget.

Streamlit: Python to Web App in Minutes

What is Streamlit?

A Python library that turns scripts into interactive web applications.

If your Python script is a recipe,
Streamlit is the kitchen that serves it to customers

No Need to Learn:

  • No HTML
  • No CSS
  • No JavaScript
  • No web frameworks

Just Python!

# Installation (run once) pip install streamlit # Create app.py file import streamlit as st import pandas as pd st.title('My First Dashboard') st.write('Hello, World!') # Run from terminal: # streamlit run app.py
How it works: Streamlit re-runs your entire script from top to bottom every time user interacts with a widget.

Streamlit: Basic Building Blocks

import streamlit as st import pandas as pd import plotly.express as px # Title and text st.title('Budget Dashboard') st.header('Overview') st.subheader('Key Metrics') st.write('This is regular text') # Load and display data df = pd.read_csv('budget.csv') st.dataframe(df) # Interactive table st.table(df.head()) # Static table # Display metrics st.metric( label='Total Budget', value='$1.2M', delta='+15%' ) # Display Plotly chart fig = px.bar(df, x='dept', y='budget') st.plotly_chart(fig)

Display Functions

FunctionUse For

st.dataframe()Interactive table
st.metric()KPI with delta

Pro Tip: st.write() is smart! It detects DataFrames, figures, and markdown automatically.

Streamlit Widgets: User Input

Widgets are the controls that let users interact with your dashboard

selectbox

Choose ONE option from a list

st.selectbox('Label', options)

Use for: Choosing a department, year, category

multiselect

Choose MULTIPLE options

st.multiselect('Label', options)

Use for: Selecting several departments to compare

slider

Choose a number in a range

st.slider('Label', min, max)

Use for: Budget range, year range, thresholds

checkbox

Yes/No toggle

st.checkbox('Label')

Use for: Show/hide data, include/exclude options

Widgets RETURN values. Store them in variables to use in your code!

CONTENT

Widget: Selectbox

import streamlit as st import pandas as pd import plotly.express as px # Load data df = pd.read_csv('budget.csv') # Create dropdown department = st.selectbox( 'Select Department:', options=df['department'].unique() ) # Filter data based on selection filtered = df[df['department'] == department] # Show filtered results st.write(f'Showing data for: {department}') st.dataframe(filtered) # Chart updates automatically! fig = px.bar(filtered, x='project', y='budget') st.plotly_chart(fig)

How It Works

  1. User selects "Education"
  2. Streamlit re-runs entire script
  3. department variable = "Education"
  4. Filter applies to DataFrame
  5. Chart shows only Education data
Common Mistake: Forgetting that selectbox RETURNS a value. Always store it!

Bad: st.selectbox(...)
Good: choice = st.selectbox(...)

Widget: Multiselect

# Multiselect returns a LIST departments = st.multiselect( 'Select Departments:', options=df['department'].unique(), default=['Health', 'Education'] # Pre-selected ) # Handle empty selection if len(departments) == 0: st.warning('Please select at least one department') else: # Filter by list of values filtered = df[df['department'].isin(departments)] st.write(f'Comparing: {", ".join(departments)}') # Compare departments fig = px.bar( filtered, x='department', y='budget', color='department' ) st.plotly_chart(fig)

Selectbox vs Multiselect

selectbox Returns: string
"Education"
multiselect Returns: list
["Education", "Health"]
Filtering Tip: Use .isin() for list filtering:

df[df['col'].isin(selected_list)]

Widget: Slider

# Single value slider min_budget = st.slider( 'Minimum Budget (COP)', min_value=0, max_value=1000000000, value=100000000, # Default step=10000000 ) # Range slider (two handles) budget_range = st.slider( 'Budget Range', min_value=0, max_value=1000000000, value=(100000000, 500000000) # Tuple = range ) # Filter with range filtered = df[ (df['budget'] >= budget_range[0]) & (df['budget'] <= budget_range[1]) ] st.write(f'Showing {len(filtered)} projects')

Slider Variants

Single value:

value=100 (integer)

Returns: 100

Range:

value=(100, 500) (tuple)

Returns: (100, 500)

Also Available:
st.select_slider() - Slider with discrete options (categories)

Putting It Together: Complete Dashboard

import streamlit as st import pandas as pd import plotly.express as px # Page config st.set_page_config(page_title='Budget Dashboard', layout='wide') # Load data df = pd.read_csv('budget_execution.csv') # SIDEBAR FILTERS with st.sidebar: st.title('Filters') departments = st.multiselect('Departments', df['department'].unique(), default=df['department'].unique()) year_range = st.slider('Year', 2020, 2024, (2020, 2024)) show_table = st.checkbox('Show data table') # APPLY FILTERS mask = (df['department'].isin(departments)) & (df['year'].between(*year_range)) filtered = df[mask] # MAIN CONTENT st.title('Budget Execution Dashboard') # Metrics row col1, col2, col3 = st.columns(3) col1.metric('Total Budget', f"${filtered['approved'].sum():,.0f}") col2.metric('Executed', f"${filtered['executed'].sum():,.0f}") col3.metric('Execution Rate', f"{(filtered['executed'].sum()/filtered['approved'].sum()*100):.1f}%") # Charts st.plotly_chart(px.bar(filtered, x='department', y='executed', color='year', barmode='group')) if show_table: st.dataframe(filtered)

Running Your Streamlit App

Step-by-Step

  1. Save your code as app.py
  2. Open terminal in that folder
  3. Run: streamlit run app.py
  4. Browser opens at localhost:8501
  5. Edit code, save, app reloads!
Shortcut: Press R in the browser to rerun, or enable "Always rerun" in Settings menu.
# Terminal commands # Navigate to your project folder cd /path/to/your/project # Run the app streamlit run app.py # Run with specific port streamlit run app.py --server.port 8080 # Run in headless mode (no browser) streamlit run app.py --server.headless true
