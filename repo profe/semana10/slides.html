Key Principle: Do as much as possible in Power Query

Power Query transformations are applied every time data refreshes. This means your cleaning is repeatable and automated.

Like a recipe that you follow every time - not just cooking once and hoping for the best.

Power Query: Common Operations

These are the "prep steps" you'll use most often

Operation Python Equivalent How in Power Query
Remove columns df.drop(columns=[...]) Right-click column header → Remove
Filter rows df[df['x'] > 10] Click filter arrow → Number Filters
Change data type df['x'].astype(int) Click column type icon → Choose type
Replace values df['x'].replace(...) Transform → Replace Values
Split column df['x'].str.split(...) Transform → Split Column
Merge tables pd.merge(df1, df2, ...) Home → Merge Queries
Pro Tip: Every step you take is recorded in "Applied Steps" on the right panel. You can delete, reorder, or modify steps anytime.

DAX: Formulas on Steroids

DAX = Data Analysis Expressions

The formula language of Power BI. Like Excel formulas, but more powerful and context-aware.

Excel: =SUM(A1:A10) works on fixed cells
DAX: SUM(Sales[Amount]) works on the ENTIRE column, but respects filters

The 4 Essential DAX Functions

SUM(Table[Column])

Adds up all values in a column

AVERAGE(Table[Column])

Calculates the mean

COUNT(Table[Column])

Counts non-blank values

DIVIDE(Numerator, Denominator, 0)

Safe division (handles divide by zero)

DAX in Practice: Birth Records

Real measures you'll create for our dataset

Total Births

Total Births = COUNTROWS(NacidosVivos)

Counts total number of birth records

Average Birth Weight

Avg Birth Weight = AVERAGE(NacidosVivos[peso_nacimiento])

Average weight across all filtered records

Male Percentage

Male % = DIVIDE( COUNTROWS(FILTER(NacidosVivos, NacidosVivos[sexo] = "M")), COUNTROWS(NacidosVivos), 0 )

Safe percentage calculation with DIVIDE

CRITICAL: Measures vs Columns

This is the #1 source of Power BI confusion!

Understand this now and save hours of debugging later.

Calculated Column

  • 💾 Stored in the data model
  • ⏱ Calculated once when data loads
  • 📎 Uses row context (one row at a time)
  • 📈 Increases file size
  • 🎯 Use for: filtering, slicing, grouping
Age Group = IF([age] < 20, "Teen", "Adult")

Measure

  • 💫 NOT stored - calculated on demand
  • ⚡ Calculated every time you interact
  • 📊 Uses filter context (respects slicers)
  • 🐱 No impact on file size
  • 🎯 Use for: aggregations, KPIs, percentages
Total Sales = SUM(Sales[Amount])

Rule of Thumb

Need to aggregate (SUM, AVG, COUNT)? → Use a Measure
Need a new category to filter by? → Use a Calculated Column

Power BI vs Python: When to Use Each

Both are powerful. Choose based on the situation.

Use Power BI When...

  • ✅ Quick dashboard needed (hours, not days)
  • ✅ Business users will interact with it
  • ✅ Data needs to refresh automatically
  • ✅ Sharing with non-technical stakeholders
  • ✅ Standard visualizations are sufficient
  • ✅ Integration with Microsoft ecosystem

Use Python When...

  • ✅ Complex data transformations needed
  • ✅ Machine learning or statistical analysis
  • ✅ Custom visualizations required
  • ✅ Automation of entire data pipeline
  • ✅ Reproducible research (notebooks)
  • ✅ Version control with Git

The Best Analysts Use BOTH

Python for heavy lifting (cleaning, ML) → Power BI for final presentation

Dashboard Layout Best Practices

DO This

  • ✅ KPIs/Cards at the top
  • ✅ Slicers on left or top
  • ✅ Most important chart = largest
  • ✅ Consistent color scheme
  • ✅ Clear titles on every visual
  • ✅ White space between elements

AVOID This

  • ❌ Too many visuals (max 6-8)
  • ❌ Rainbow of colors
  • ❌ 3D charts (ever)
  • ❌ Pie charts with many slices
  • ❌ Overlapping elements
  • ❌ Tiny fonts

Common Mistakes to Avoid

Mistake #1: Using COUNT instead of COUNTROWS

COUNT only works on columns with values. COUNTROWS counts all rows in a table.

Wrong: COUNT(Table[Column])
Right: COUNTROWS(Table)

Mistake #2: Dividing without DIVIDE

Regular division crashes on divide-by-zero. DIVIDE handles it gracefully.

Wrong: [A] / [B]
Right: DIVIDE([A], [B], 0)

Mistake #3: Creating Column when needing Measure

If you want a value that changes with filters, use a MEASURE, not a calculated column.

Column: Static, calculated once
Measure: Dynamic, respects filters

Mistake #4: Skipping Power Query

Trying to clean data with DAX is painful. Do it in Power Query where it's visual and repeatable.

Clean data = Power Query
Calculations = DAX
